# 决策验证机制

**目的：** 记录每次交易决策，次日自动验证准确性，持续提高 AI 决策能力

---

## 📊 工作流程

```
T 日 14:00 ──▶ 生成决策 ──▶ 保存记录 (decision_records/YYYY-MM-DD.json)
     ↓
T+1 日 09:00 ──▶ 预检管线 ──▶ 验证昨日决策
     ↓
计算准确率 ──▶ 标记已验证 ──▶ 更新知识库
```

---

### 决策记录文件

**路径：** `08-fund-daily-review/decision_records/YYYY-MM-DD.json`

## ✅ 验证逻辑

### HOLD 决策
- **正确：** 次日涨跌幅 > -2%
- **错误：** 次日涨跌幅 ≤ -2%

### BUY 决策
- **正确：** 次日涨跌幅 > 0%
- **错误：** 次日涨跌幅 ≤ 0%

### SELL 决策
- **正确：** 次日涨跌幅 < 0%
- **错误：** 次日涨跌幅 ≥ 0%

## 🎯 学习机制

### 错误分析
1. **记录错误决策** - 保存错误案例
2. **分类错误原因** - 市场误判/数据延迟/逻辑缺陷
3. **更新决策规则** - 避免重复错误

### 知识库更新
- ✅ 成功决策 → 强化相关规则
- ❌ 错误决策 → 调整权重或规则
